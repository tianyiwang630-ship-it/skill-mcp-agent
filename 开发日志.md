# 开发日志

## 2026-03-21

### 背景
这一轮重点在 `agent-alpha` 的第一阶段重构，目标是先把骨架理顺，不碰第二阶段的多 agent 编排。

### 更改内容
- 将 `Agent` 本体从单一入口里拆出，核心实例放到 `agent-alpha/agent/core/core_agent.py`，让后续单 agent 和多 agent runner 可以分开。
- 新增 `agent-alpha/agent/core/session_paths.py`，统一管理 `workspace_root/sessions/<session_id>/input|output|temp` 和 `workspace_root/logs`。
- 新增 `agent-alpha/agent/core/prompt_docs_loader.py`，让 session 级 `input/AGENTS.md` 和 `input/SOUL.md` 可以自动进入 system prompt。
- 调整 `agent-alpha/agent/core/system_prompt_builder.py`，把 workspace、session、logs、skills、mcp 路径一起注入 prompt。
- 将 skill 加载改成两层模式：启动时只读 `SKILL.md` 的摘要，正文按需加载，不再走“一个 skill 一个 tool”的旧方式。

## 2026-03-21

### 背景
需要把官方 Playwright MCP 接进 `agent-alpha`，同时满足这几个要求：默认有头、浏览器装全局、状态留本地、以后可切无头共享登录态。

### 更改内容
- 在 `agent-alpha/mcp-servers/playwright/` 下建立本地 wrapper，而不是直接依赖运行时 `npx` 临时拉取。
- 增加 `mcp.config.json`、`playwright.headed.config.json`、`playwright.headless.config.json` 和 `server.js`，形成“一个目录、双配置”的结构。
- 约定默认走有头模式，使用持久 profile；无头模式预留为共享 `storage-state` 读取。
- 状态目录固定在 `agent-alpha/mcp-servers/playwright/state/` 下，分为 `profiles/` 和 `storage/`。
- 新增 `agent-alpha/mcp-servers/playwright/README.md`，把“浏览器装全局、状态留本地、关闭前同步 storage-state”的规则写清楚。
- 增加对应测试，确认 Playwright MCP 的目录结构、配置关系和扫描行为正常。

## 2026-03-22

### 背景
原来的 LLM 配置只有一套 `LLM_BASE_URL / LLM_API_KEY / LLM_MODEL_NAME`，已经不适合一个项目里接多家 OpenAI 兼容模型，也不利于后面多 agent 使用不同模型组。

### 更改内容
- 新增 `agent-alpha/config/llm_profiles.json`，将模型配置改为 profile 方式管理。
- 新增 `agent-alpha/agent/core/llm_profiles.py`，负责读取 profile、解析默认项，并从环境变量中取真实 API key。
- 修改 `agent-alpha/agent/core/llm.py`，`LLMClient` 不再直接读取全局常量，而是基于 profile 创建。
- 修改 `agent-alpha/agent/core/core_agent.py`，支持创建 `Agent` 时传入 `llm_profile_name`。
- 保持 `ContextManager` 接口不变，它仍然只接收一个 `llm` 实例，这样以后如果想单独给总结压缩切模型，只需要在 `Agent` 层调整。
- 删除旧的全局 LLM 直连常量，不再保留旧入口，避免两套配置方式并存。

## 2026-03-22

### 背景
这一轮继续整理 `agent-alpha` 的职责边界，目标是让 `Agent` 只负责在一个现成工作目录里运行，把 session 创建、日志落盘这些 runner 级职责从 `Agent` 身上拿掉。

### 更改内容
- 调整 `agent-alpha/agent/core/core_agent.py`，`Agent` 现在只接收外界传入的 `workspace_root`，并自动补齐 `input`、`output`、`temp` 三个目录。
- `Agent` 不再自己生成 `session_id`、不再维护 `logs_dir`、不再负责写 session log；改为提供 `get_session_log_data()`，只暴露日志内容。
- 重写 `agent-alpha/agent/core/session_paths.py`，拆成两类能力：一类给 `Agent` 保证工作目录结构，一类给 CLI/session runner 创建 `sessions/<id>` 和 `logs`。
- 调整 `agent-alpha/agent/core/main.py`，把 session 创建、日志文件名生成、日志写盘、session 索引更新都收回到 `main.py`。
- 重写 `agent-alpha/agent/core/system_prompt_builder.py` 的路径区块，按“当前工作区 / 系统资源区 / 运行记录区”分组，让模型更清楚哪些目录是当前任务目录，哪些是 skill/MCP 资源，哪些是日志记录。

## 2026-03-22

### 背景
继续收束 `Agent` 的 workspace 模型。为了支持单 agent 和以后多 agent 的不同协作方式，这一轮不再把 workspace 固定成 `input/output/temp` 模板，而是改成“一个或多个普通 workspace”，并且只从第一个 workspace 根目录读取人格和规则文档。

### 更改内容
- 修改 `agent-alpha/agent/core/core_agent.py`，`Agent` 现在支持接收一个或多个 workspace；若传多个，只把第一个当作私有/专属 workspace，其他作为额外可工作的 workspace。
- 修改 `agent-alpha/agent/core/prompt_docs_loader.py`，只从第一个 workspace 根目录读取 `AGENTS.md` 和 `SOUL.md`，不再扫描嵌套子目录，也不再从 `input` 目录读取。
- 修改 `agent-alpha/agent/core/system_prompt_builder.py`，路径区块改成“Private Workspace / Additional Workspaces / System Resource Paths / Runtime Records”，并在规则里明确说明：多 workspace 的分工主要由 `AGENTS.md` 解释。
- 修改 `agent-alpha/agent/core/main.py` 和 `agent-alpha/agent/core/session_paths.py`，CLI 会话目录不再自动创建 `input/output/temp`，只创建会话目录本身和日志目录。
- 同步调整测试，确认多 workspace、文档识别范围和日志索引行为符合新模型。
